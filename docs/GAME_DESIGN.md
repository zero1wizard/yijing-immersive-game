# GAME DESIGN

## 产品定位

易境把《易经》阅读改造成一种“在局中学习”的体验。用户不是被动查释义，而是先进入情境，做出一个动作，再看动爻、转卦与后果如何改变理解。产品目标不是预测结果，而是训练用户对局势、关系、节奏和边界的判断。

## 目标用户

- 对《易经》感兴趣，但觉得原典直接进入门槛较高的初学者
- 已经读过部分材料，希望用交互方式建立“卦与情境”的直觉联系的用户
- 想把每天的反思、笔记和阅读练习收拢到一个工具里的长期学习者

## 核心体验承诺

- 进入快：首次打开就能直接遭遇一个卦境
- 理解快：几分钟内明白“读情境 -> 选动作 -> 看转卦”的循环
- 回访强：每日一卦、札记和日志让用户愿意重复回来

## 核心循环

1. 进入一个卦境，或抽取今日一卦。
2. 阅读当前情境设置、主题张力和六爻提示。
3. 在三种行动姿态中做选择：观势、推进、退整。
4. 看见动爻、转卦与后果，理解当前动作如何改变局面。
5. 解锁相关卦、灵感条目与个人记录。
6. 在卦典和札记中回看，逐步形成自己的判断谱系。

## 世界结构

- 起势：开局、萌芽、待时、初次登场
- 处世：关系、边界、礼度、感应与差异
- 应变：冲突、险境、撤退、修补、变革
- 治理：组织、秩序、制度、公共资源、群体协调
- 修身：节制、定静、涵养、诚信、内在功课
- 终局：丰盛、困局、回转、未竟与重新开始

六个章节不是原典分篇，而是产品为了可游玩性建立的“理解地图”。它让用户先通过主题进入，再反向建立对卦象的认识。

## 界面结构

- 左侧英雄面板：品牌、入口按钮、进度统计、章节地图、今日反思
- 右侧主内容区：在“当前卦境”和“卦典检索”之间切换
- 当前卦境页：情境、选择、静读材料、札记、最近推演
- 卦典页：搜索、筛选、全部 64 卦概览与跳转

## MVP 范围

- 单页应用，不做账号系统
- 本地持久化保存个人进度与札记
- 全量 64 卦内容入口，文本为原创精简版
- 三种统一动作姿态，保证玩法一致性
- 以“反思型日课”替代迷信式占验呈现

## 成功标准

- 第一次进入就能马上开始玩
- 三分钟内理解“遭遇卦境并做选择”的玩法
- 一周内愿意重复回来写札记、查相关卦、完成日课
- 用户能在日志中看到自己对同类主题的判断变化

## 非目标

- 不做算命式结果承诺
- 不做复杂账号、云同步和社区功能
- 不在 MVP 阶段提供权威注本替代阅读
